# 创建一个 Vue 项目

## 使用 vue create 命令

要创建一个新项目，可以运行以下命令：

```bash
vue create hello-world
```

------

【注意事项】

这将启动一个交互式的命令行界面，指导您逐步配置和创建项目。

如果您在 Windows 系统上通过 Git Bash（使用 minTTY）运行该命令，可能会遇到交互提示符无法正常工作的情况。为了解决这个问题，您可以通过以下命令来启动项目创建：

```bash
winpty vue.cmd create hello-world
```

另外，您还可以为 `vue` 命令创建一个别名，以便在以后更方便地使用，在 `~/.bashrc` 文件中添加以下行：

```bash
alias vue='winpty vue.cmd'
```

保存文件后，重新启动 Git Bash 终端会话以使更改生效。

------

在项目创建过程中，您将被提示选择一个预设（preset）。Vue CLI 提供了以下选项：

- **默认预设**：包含基本的 Babel + ESLint 配置，适合快速启动一个新项目。
- **手动选择特性**：允许您根据项目的需求手动选择特性，例如 Vue Router、Vuex、TypeScript 等。这种方式适合生产环境下的项目，提供了更多的灵活性。

如果您选择了手动配置，在操作的最后，您还可以选择将当前配置保存为一个预设（preset），以便将来复用。保存的预设会存储在用户的 home 目录下的 `.vuerc` 文件中，您可以随时编辑这个文件来修改保存的预设和选项。

`vue create` 命令还提供了多种可选参数，以进一步定制项目创建过程。您可以通过运行以下命令查看这些选项：

```bash
vue create --help
```

常用选项包括：

- **`-p, --preset <presetName>`**：使用已保存的或远程的预设选项，跳过交互提示。
- **`-d, --default`**：使用默认预设，跳过交互提示。
- **`-m, --packageManager <command>`**：指定使用的包管理器（如 npm 或 yarn）。
- **`-r, --registry <url>`**：使用指定的 npm registry 安装依赖。
- **`-g, --git [message]`**：初始化 Git 仓库，并可选指定初始化提交信息。
- **`-n, --no-git`**：跳过 Git 初始化。
- **`-f, --force`**：覆写已存在的目标目录。
- **`-b, --bare`**：创建项目时省略默认组件中的新手指导信息。

## 使用图形化界面创建项目

如果您更喜欢图形化界面，Vue CLI 提供了 `vue ui` 命令，通过该命令，您可以在浏览器中以图形化界面创建和管理项目：

```bash
vue ui
```

该命令会打开一个浏览器窗口，引导您完成项目创建的流程。

## 使用旧版本模板（2.x）

如果您需要使用 Vue CLI 2.x 的 `vue init` 功能，您可以安装一个桥接工具：

```bash
npm install -g @vue/cli-init
```

然后，您可以使用以下命令创建项目，效果与 `vue-cli@2.x` 相同：

```bash
vue init webpack my-project
```

这为那些需要使用旧版模板的开发者提供了兼容性支持。
