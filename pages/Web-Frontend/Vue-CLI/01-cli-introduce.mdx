# Vue CLI 基本介绍

## Vue CLI 简介

Vue CLI 是一个基于 Vue.js 的完整开发系统，旨在帮助开发者快速启动和管理 Vue.js 项目。它提供了一整套工具和功能，使开发过程更加高效和便捷，具体包括以下几个方面：

1. **交互式项目脚手架**：通过 `@vue/cli` 实现，Vue CLI 提供了一个命令行工具，支持交互式地创建和配置新项目。开发者只需通过简单的命令，即可搭建一个完整的 Vue.js 项目。
2. **零配置的原型开发**：通过 `@vue/cli` 与 `@vue/cli-service-global` 的结合，Vue CLI 允许开发者进行零配置的原型开发。这意味着你可以快速搭建原型，而不必在项目初期花费大量时间在配置文件上。
3. **运行时依赖**：Vue CLI 提供了一个可升级的运行时依赖包 `@vue/cli-service`，它基于 webpack 构建，拥有合理的默认配置。开发者可以通过项目内部的配置文件进行自定义配置，或者通过插件系统进行功能扩展。
4. **丰富的官方插件集合**：Vue CLI 集成了前端开发生态中众多优质工具，通过官方插件集合，开发者可以方便地在项目中加入 Babel、TypeScript、ESLint、单元测试和端到端测试等功能。
5. **图形化用户界面**：Vue CLI 提供了一个完全图形化的用户界面，使得创建和管理 Vue.js 项目更加直观和便捷，特别适合那些不习惯命令行操作的开发者。

Vue CLI 的设计目标是标准化 Vue 生态中的工具链。它确保了各种构建工具可以基于智能的默认配置无缝衔接，减少了开发者在配置上的困扰，让他们能够专注于应用的开发。同时，Vue CLI 也保留了高度的配置灵活性，开发者无需退出项目即可调整各项工具配置。

## Vue CLI 系统组件

Vue CLI 由多个独立的模块组成，这些模块在同一个仓库中管理并分别发布。

### CLI

`@vue/cli` 是一个全局安装的 npm 包，提供了在终端中使用的 `vue` 命令。通过该命令，开发者可以使用 `vue create` 快速生成新项目，或使用 `vue serve` 构建新想法的原型。此外，还可以通过 `vue ui` 命令启动一个图形化界面来管理所有项目。接下来的指南将详细介绍每个功能的使用方法。

### CLI 服务

`@vue/cli-service` 是每个由 `@vue/cli` 创建的项目中局部安装的开发环境依赖包。它建立在 webpack 和 webpack-dev-server 之上，提供了：

- 加载其他 CLI 插件的核心服务；
- 针对大多数应用优化的内部 webpack 配置；
- 项目内部的 `vue-cli-service` 命令，支持 `serve`、`build` 和 `inspect` 等命令。

如果你熟悉 React 的 `create-react-app`，可以将 `@vue/cli-service` 类比为 `react-scripts`，尽管它们的功能集合有所不同。CLI 服务章节将详细介绍其具体用法。

### CLI 插件

CLI 插件是向 Vue 项目提供额外功能的 npm 包，例如 Babel/TypeScript 转译、ESLint 集成、单元测试和端到端测试等。Vue CLI 插件通常以 `@vue/cli-plugin-`（官方插件）或 `vue-cli-plugin-`（社区插件）开头，便于识别和使用。

当你在项目中运行 `vue-cli-service` 命令时，CLI 服务会自动解析并加载 `package.json` 中列出的所有 CLI 插件。这些插件可以在项目创建过程中一并安装，或者在项目后期加入，并且可以组合成一组可复用的预设（preset）。我们将在插件和预设章节中进行更深入的探讨。
